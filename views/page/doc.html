Introduction

Thai Language API เป็น RESTful API ที่ให้บริการเกี่ยวกับภาษา ซึ่งจะส่งค่ากลับมาอยู่ในรูปแบบของ JSON สำหรับทุกๆครั้งที่มีการร้องขอ API

Authentication

การยืนยันตัวตนผ่านสามารถทำได้โดยการแนบ Token มากับ Header ของHTTP request

API Response

เมื่อมีการส่ง HTTP Request ไป ก็จะต้องมีการส่ง HTTP response code เพื่อแสดงให้เห็นว่าการrequest นั้นสำเร็จหรือไม่
Status 200 หมายถึงการร้องขอ API สำเร็จ Status 401 หมายถึงการร้องขอ API นั้นไม่มีการแนบ authentication token หรือ authentication token นั้นไม่ถูกต้อง

Endpoints

    หากต้องการ request ที่ระบุข้อมูลที่ต้องการสามารถทำได้โดยเพิ่มชนิดของข้อมูล(meaning , synonym , frequency)ไปในurl

  • GET words/:name

    • การ request ที่ไม่มีการระบุเฉพาะจะได้รับข้อมูลที่คืนกลับมาได้แก่ Part of speech, Prononciation, Meaning, Frequency และ Tag

  • GET words/meaning/:name

    • จะได้ข้อมูลคืนมาได้แก่ Part of Speech และ Meaning

  • GET words/synonym/:name

    • จะได้ข้อมูลคืนมาได้แก่ Synonym คือคำไวพจน์ของคำนั้นๆ

  • GET words/frequency/:name

    • จะได้ข้อมูลคืนมาได้แก่ frequency คือความถี่ที่คำนั่นๆถูกใช้งานบน twitter โดยเปรียบเทียบต่อ 1ล้านคำ

  • GET words/example/:name

    • ข้อมูลที่ได้จะเป็นตัวอย่างของประโยคที่มีคำนั้นๆ เพื่อเป็นตัวอย่างการใช้งานของคำนั้นๆ

API ENDPOINT

htttp://thailanguageapi.com/api

Authentication Example

EXAMPLE REQUEST
curl -H "x-access-token: [ACCESSTOKEN]" http://thailanguageapi.com/api/words/นอน

Outputs Example

Status 200 Success - ok
Status 401 Unauthorized - Failed to authenticate token or No token provided in request
Status 404 Not Found - Wrong url
